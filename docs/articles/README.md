## Conventions

- **Folder naming**: `YYYY-MM-kebab-slug` — date is intended publish month, slug is short and descriptive.
- **Authoring source**: write in Markdown (`article.md`); paste into Substack at publish time.
- **Assets**:
  - `assets/diagrams/` — Excalidraw `.excalidraw` (source) + `.png` (export, 2× scale, white background). Always commit both — the JSON source lets you (and others) edit the diagram later.
  - `assets/plots/` — matplotlib PNGs at 300 DPI, generated by scripts in `scripts/`
  - `assets/videos/` — MP4s under ~10 MB; larger ones go in git LFS or external store
- **Reuse across articles**: copy assets between folders rather than cross-referencing. `_shared/` is reserved for assets that are intentionally identical in every article.
- **At publish time**: tag the commit with `git tag -a article-YYYY-MM-slug -m "Published"`. Gives readers a permalink to the exact codebase the article describes.
